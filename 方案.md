# MBTI 聊天分析 — 完整产品方案

## Context

市面上的 MBTI 测试产品几乎都是「答题模式」——用户面对几十道选择题，体验枯燥且不自然。没有人真的喜欢做问卷。本项目的核心洞察：**通过自然对话来推断人格类型，比填问卷更准确，因为人在聊天中的真实反应比刻意选择更能反映性格。**

分三个阶段交付：
1. **Phase 1 — Skill 体验版**：Claude Code / Codex slash command，终端内即可体验
2. **Phase 2 — Web 应用**：完整前后端，可分享、可回看
3. **Phase 3 — APP 应用**：移动端原生体验

---

## Phase 1：Skill 体验版

### 目标
在 Claude Code 和 Codex 中通过 `/mbti-analysis` 启动一段自然对话，8-12 轮后输出完整 MBTI 分析报告。

### 核心设计

#### 对话三阶段

| 阶段 | 轮次 | 风格 | 目的 |
|------|------|------|------|
| **破冰暖场** | 1-4 | 轻松生活场景假设 | 建立信任，同时广撒网探测 4 个维度 |
| **深度追问** | 5-9 | 根据前面回答针对性追问 | 对信号不足的维度深入挖掘 |
| **校准收尾** | 10-12 | 反转提问、自我认知确认 | 交叉验证，解决矛盾信号 |

#### MBTI 四维度探测策略

**E（外向）vs I（内向）— 能量来源**
- 破冰场景：「想象你刚结束一个超累的项目，周末你会选择怎么恢复精力？」
- 深入追问：「你觉得一个人待着和跟朋友在一起，哪种状态你更能产出好想法？」
- 信号词：E→聚会、聊天、出去、热闹；I→独处、安静、充电、自己

**S（感觉）vs N（直觉）— 信息收集**
- 破冰场景：「如果让你从零开始学一样完全没接触过的东西，你会怎么开始？」
- 深入追问：「做决定的时候，你更相信过往经验还是直觉判断？」
- 信号词：S→细节、步骤、实际、经验；N→可能性、大局、灵感、未来

**T（思考）vs F（情感）— 决策方式**
- 破冰场景：「好朋友做了一个你觉得明显不靠谱的决定，你会怎么做？」
- 深入追问：「团队里有人能力不行但态度很好，裁员名单上你会怎么考虑？」
- 信号词：T→逻辑、公平、效率、分析；F→感受、和谐、价值观、同理

**J（判断）vs P（感知）— 生活方式**
- 破冰场景：「下周要出去旅行，你现在会做什么准备？」
- 深入追问：「临时计划变动对你来说是惊喜还是压力？」
- 信号词：J→计划、清单、提前、确定；P→随性、灵活、临时、可能

#### 内部评分机制（用户不可见）

每轮对话后，AI 内部维护一个评分状态：
```
维度评分 = {
  E_I: { E: 0, I: 0, confidence: "low" },
  S_N: { S: 0, N: 0, confidence: "low" },
  T_F: { T: 0, F: 0, confidence: "low" },
  J_P: { J: 0, P: 0, confidence: "low" }
}
```
- 每个回答提取 0-3 个信号，每个信号有强度：强(3)、中(2)、弱(1)
- 置信度：<60% 边界型、60-75% 中等、75-90% 明确、>90% 强烈
- 下一个问题优先探测置信度最低的维度

## Phase 2：Web 应用

### 技术栈
- **前端**：Next.js（React）+ TailwindCSS + Framer Motion
- **后端**：Python FastAPI（与用户技术栈一致）
- **数据库**：SQLite（初期）→ PostgreSQL（扩展）
- **AI 接口**：Google Gemini API（gemini-2.5-flash），通过 Google AI SDK 接入

### 架构设计

│       └────────────┼────────────┘       │

### 核心功能
1. **聊天界面**：仿微信/iMessage 气泡样式，打字动画，让人有聊天的感觉
2. **实时评分**：后端维护 session 状态，每轮对话实时更新四维度评分
3. **动态报告**：生成可视化报告页，含雷达图、维度条、认知功能栈
4. **分享链接**：每份报告生成唯一短链，可分享到社交平台
5. **历史记录**：可回看完整对话和分析过程
6. **匿名模式**：无需注册即可使用，可选绑定账号保存记录

### API 设计
```
POST   /api/session/create      # 创建新对话
POST   /api/session/{id}/chat   # 发送消息（SSE 流式返回）
GET    /api/session/{id}/status  # 获取当前评分状态
POST   /api/session/{id}/finish # 结束对话，生成报告
GET    /api/report/{id}          # 获取报告
GET    /api/report/{id}/share    # 获取分享页面
```

### 数据模型
```python
# Session - 对话会话
session_id, created_at, status, mbti_result, confidence_scores

# Message - 对话消息
message_id, session_id, role, content, signals_extracted, created_at

# Report - 分析报告
report_id, session_id, mbti_type, dimension_scores, 
cognitive_stack, strengths, growth_areas, share_token
```

## Phase 3：APP 应用

### 技术选型
- **React Native**（Expo）：一套代码 iOS + Android，复用 Web 端组件逻辑
- 后端直接复用 Phase 2 的 FastAPI

### 额外功能
1. **人格成长追踪**：定期重新测试，查看变化趋势
2. **好友对比**：邀请好友测试后，查看性格匹配度和沟通建议
3. **AI 性格教练**：基于 MBTI 类型提供个性化成长建议（持续对话）
4. **每日一问**：轻量级日常互动，持续优化人格画像
5. **Push 通知**：每日心理小贴士，基于用户 MBTI 类型定制
